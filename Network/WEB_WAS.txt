- 웹 브라우저와 웹 서버의 통신과정
클라이언트(Client): 일반적으로 네트워크 프로그램에서 요청하는 쪽
서버(Server): 요청을 받아 알맞은 기능이나 데이터를 제공하는 쪽

url에서 프로토콜이 http인 경우 별도로 포트를 지정하지 않으면 80번 포트로 연결한다는 것을 뜻함
ssl 적용하여 프로토콘 https일 경우 443포트 사용

Web was

Web server: 클라이언트로부터 요청을 받아 요청한 것을 넘겨주는 일
404 not found: 요청한 자료를 서버에서 찾지 못함
보통 외부 사용자들에 대응하기 위해 DMZ존에 위치
정적 웹 페이지를(html, css, 이미지, javascript파일) 내보내는 역할
--webtob이슈정리
--html, css가 적용되지 않은 페이지가 나오는 상황이 있었는데
해당 WebtoB 경로에 소스가 올라가있지 않은 것이 원인이였음
해당 WebtoB 경로는 환경설정파일 http.m을 확인
DocRoot에 소스를 올려주면 정적 웹페이지 작동하는 것을 확인할 수 있음
--80포트 접속시에 화면이 안나오는 상황이 있었는데
http.m파일 확인시 *VHOST의 해당 서버에 PORT가 등록되지 않아 그런거였음
--신규서버 사용 시에 방화벽 오픈 외로 Window 서버 담당자에게 따로 80포트를 오픈해달라고 요청해야함
해당 서버 포트에 80 추가 후 정상 작동되는것을 확인
